# Context Component

## Назначение

Компонент `context` предоставляет контекст выполнения для конвейеров обработки запросов, управляя состоянием сессии и передачей сообщений между компонентами.

## Основные файлы

- `context.hpp/cpp` - Определение контекста выполнения

## Ключевые классы и функциональность

- **`context_t`** - Контекст выполнения конвейера, содержащий:
  - `session_id` - идентификатор сессии
  - Actor addresses - адреса акторов для взаимодействия
  - Query parameters - параметры запроса
  - Возможности отправки сообщений через фреймворк actor-zeta

### Основные возможности

- Управление состоянием сессии во время выполнения запросов
- Передача сообщений между компонентами системы
- Интеграция с хранилищем параметров
- Связь с актор-моделью для асинхронной обработки

## Использование

Контекст передается через весь конвейер обработки запросов, обеспечивая доступ к необходимым ресурсам и состоянию сессии на каждом этапе выполнения.
