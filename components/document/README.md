# Document Component

## Назначение

Компонент `document` реализует документо-ориентированную модель данных с поддержкой JSON, обеспечивая операции с документами в стиле MongoDB.

## Основные файлы

- `document.hpp/cpp` - Основной класс документа
- `document_id.hpp` - Идентификаторы документов
- `value.hpp/cpp` - Типизированные значения
- `base.hpp` - Базовые определения
- `json_trie_node.hpp/cpp` - Эффективное представление JSON структур
- `string_splitter.hpp/cpp` - Утилиты для разбора строк

## Ключевые классы и функциональность

### Основные классы

- **`document_t`** - Главный класс документа с поддержкой JSON Pointer:
  - Операции чтения: `get`, `is_exists`
  - Операции записи: `set`, `remove`
  - Операции перемещения: `move`, `copy`
  - Проверка типов: `is_null`, `is_bool`, `is_int`, `is_string`, `is_array`, `is_dict`
  - Извлечение значений с проверкой типов
  - Сравнение и слияние документов
  - Сериализация/десериализация JSON

- **`json_trie_node`** - Эффективное представление JSON структуры на основе trie

- **`value_t`** - Обертка для типизированных значений

### Поддерживаемые типы данных

- Примитивные типы: null, bool, int, uint, double, string
- Структурированные типы: array, dict (объект)
- Бинарные данные

### Основные операции

- Навигация по документу с использованием JSON Pointer
- Глубокое копирование и слияние
- Валидация структуры
- Преобразование в/из JSON

## Использование

Документы используются для хранения и обработки данных в документо-ориентированном хранилище, обеспечивая гибкую схему и поддержку вложенных структур.
