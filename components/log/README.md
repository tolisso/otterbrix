# Log Component

## Назначение

Компонент `log` предоставляет инфраструктуру логирования на основе библиотеки spdlog.

## Основные файлы

- `log.hpp/cpp` - Реализация логгера

## Ключевые классы и функциональность

### Основной класс

- **`log_t`** - Обертка над spdlog с функциональностью:
  - Инициализация логгера
  - Управление уровнями логирования
  - Форматированный вывод сообщений

### Уровни логирования

- **trace** - детальная трассировка выполнения
- **debug** - отладочная информация
- **info** - информационные сообщения
- **warn** - предупреждения
- **err** - ошибки
- **critical** - критические ошибки
- **off** - логирование отключено

### Основные возможности

- Шаблонные функции для форматированного логирования
- Поддержка различных sink'ов (консоль, файл)
- Автоматическая ротация логов
- Thread-safe операции

## Использование

Логгер используется во всех компонентах системы для записи диагностической информации, отладки и мониторинга работы базы данных.

```cpp
// Пример использования
log_t::info("Starting query execution for session {}", session_id);
log_t::error("Failed to open file: {}", error_message);
```
