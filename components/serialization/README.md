# Serialization Component

## Назначение

Компонент `serialization` предоставляет сериализацию и десериализацию для логических планов, выражений и компонентов запросов.

## Основные файлы

- `serializer.hpp/cpp` - Классы сериализаторов
- `deserializer.hpp/cpp` - Классы десериализаторов

## Ключевые классы и функциональность

### Сериализаторы

- **`base_serializer_t`** - Базовый интерфейс сериализатора

- **`json_serializer_t`** - JSON-сериализация:
  - Человеко-читаемый формат
  - Отладка и логирование
  - Передача по текстовым протоколам

- **`msgpack_serializer_t`** - MessagePack бинарная сериализация:
  - Компактный бинарный формат
  - Эффективная передача по сети
  - Быстрая сериализация/десериализация

### Десериализаторы

- **`base_deserializer_t`** - Базовый интерфейс десериализатора

- Соответствующие десериализаторы для каждого формата

### Поддерживаемые типы

Компонент поддерживает сериализацию:

1. **Логических узлов** - все типы узлов логического плана
2. **Выражений** - сравнения, скалярные, агрегатные, и т.д.
3. **Документов** - JSON документы
4. **Параметров** - параметры запросов
5. **Метаданных** - схемы, типы, конфигурация

### Идентификация типов

- **`serialization_type`** - Перечисление для идентификации типа сериализуемого объекта:
  - Позволяет корректную десериализацию
  - Версионирование формата
  - Проверка типов при чтении

## Использование

Сериализация используется для:

1. **Передача запросов между узлами** - в распределенной системе
2. **Персистентность** - сохранение планов запросов
3. **Кэширование** - сохранение подготовленных планов
4. **Отладка** - вывод структуры запросов
5. **Logging** - журналирование выполнения запросов

```cpp
// Пример использования
json_serializer_t serializer;
auto json_data = serializer.serialize(logical_plan_node);

// Десериализация
json_deserializer_t deserializer(json_data);
auto restored_node = deserializer.deserialize_node();
```
