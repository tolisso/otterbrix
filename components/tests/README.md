# Tests Component

## Назначение

Компонент `tests` предоставляет утилиты для генерации тестовых данных, используемых в unit-тестах и интеграционных тестах системы.

## Основные файлы

- `generaty.hpp/cpp` - Генераторы тестовых данных

## Ключевые функции

### Генерация документов

- **`gen_array()`** - Генерация документов с массивами:
  - Создание массивов различных размеров
  - Различные типы элементов
  - Вложенные массивы

- **`gen_dict()`** - Генерация документов-словарей:
  - Объекты с произвольными полями
  - Различные типы значений
  - Вложенные объекты

- **`gen_id()`** - Генерация тестовых идентификаторов:
  - Уникальные ID для документов
  - Последовательные или случайные значения

- **`gen_doc()`** - Генерация полноценных тестовых документов:
  - Комбинация различных типов данных
  - Реалистичная структура
  - Настраиваемая сложность

### Генерация табличных данных

- **`gen_data_chunk()`** - Генерация колоночных тестовых данных:
  - Векторизованные данные для таблиц
  - Различные типы колонок
  - Настраиваемое количество строк
  - Генерация NULL значений

## Использование

Утилиты генерации используются в тестах для:

1. **Unit-тесты компонентов** - тестирование отдельных модулей с контролируемыми данными

2. **Интеграционные тесты** - проверка взаимодействия компонентов с реалистичными данными

3. **Performance тесты** - создание больших объемов данных для нагрузочного тестирования

4. **Edge cases** - генерация граничных случаев:
   - Пустые структуры
   - Максимальные размеры
   - Специальные значения

```cpp
// Пример использования
// Генерация тестового документа
auto doc = gen_doc(/* параметры */);

// Генерация массива документов
auto docs = gen_array(100); // 100 документов

// Генерация табличных данных
auto chunk = gen_data_chunk(schema, 1000); // 1000 строк
```

## Типичные сценарии тестирования

1. **Тестирование парсера** - документы с различной структурой
2. **Тестирование индексов** - данные с различным распределением
3. **Тестирование запросов** - наборы данных для различных типов запросов
4. **Тестирование сериализации** - документы разной сложности
5. **Stress-тесты** - большие объемы данных для проверки стабильности
