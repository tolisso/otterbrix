# Session Component

## Назначение

Компонент `session` управляет сессиями базы данных с уникальными идентификаторами для отслеживания клиентских подключений и транзакций.

## Основные файлы

- `session.hpp/cpp` - Реализация управления сессиями

## Ключевые классы и функциональность

### Основной класс

- **`session_id_t`** - Уникальный идентификатор сессии:
  - Временная метка создания
  - Инкрементальный счетчик
  - Гарантия уникальности

### Основные возможности

1. **Генерация ID сессий** - создание уникальных идентификаторов для каждого подключения

2. **Компоненты ID**:
   - Timestamp - время создания сессии
   - Counter - счетчик для различения сессий, созданных в одно время

3. **Поддержка хеширования** - для использования в hash-based контейнерах:
   - std::unordered_map
   - std::unordered_set

4. **Операции сравнения** - полный набор операторов для сравнения сессий

## Использование

Идентификаторы сессий используются для:

1. **Отслеживание клиентских подключений** - каждый клиент получает уникальную сессию
2. **Изоляция транзакций** - связывание операций с конкретной сессией
3. **Управление ресурсами** - отслеживание ресурсов, выделенных для сессии
4. **Логирование и аудит** - связывание операций с клиентами
5. **Контекст выполнения** - передача информации о сессии через конвейер обработки

```cpp
// Пример использования
auto session = session_id_t::generate();

// Использование в контексте
context_t ctx(session);
ctx.execute_query("SELECT * FROM users");
```

Сессия создается при подключении клиента и сохраняется на протяжении всего времени подключения, обеспечивая изоляцию и отслеживание операций.
