# Catalog Component

## Назначение

Компонент `catalog` управляет метаданными баз данных, схем и таблиц. Выступает в роли системного каталога, который отслеживает все объекты базы данных и их свойства.

## Основные файлы

- `catalog.hpp/cpp` - Главный класс каталога
- `schema.hpp/cpp` - Представление схемы таблицы
- `table_metadata.hpp/cpp` - Метаданные таблиц
- `table_id.hpp/cpp` - Уникальные идентификаторы таблиц
- `namespace_storage.hpp/cpp` - Управление иерархическим пространством имен
- `computed_schema.hpp/cpp` - Схемы вычисляемых/производных таблиц
- `catalog_types.hpp/cpp` - Типы данных каталога
- `catalog_error.hpp/cpp` - Обработка ошибок каталога

## Ключевые классы и функциональность

### Основные классы

- **`catalog`** - Основной класс каталога, управляющий пространствами имен и таблицами с поддержкой транзакций
- **`schema`** - Представляет схему таблицы с определениями столбцов, описаниями полей и первичными ключами
- **`table_metadata`** - Хранит метаданные о таблицах (схема, формат)
- **`table_id`** - Уникальный идентификатор для таблиц
- **`namespace_storage`** - Управление иерархическим пространством имен (database → schema → collection)
- **`computed_schema`** - Схема для вычисляемых/производных таблиц

### Основные операции

**Операции с пространствами имен:**
- Создание, удаление и перечисление баз данных, схем и коллекций

**Операции с таблицами:**
- Создание и удаление таблиц
- Переименование таблиц
- Проверка существования
- Получение метаданных

## Использование

Каталог является центральным компонентом для управления структурой базы данных и обеспечивает согласованность метаданных во всех операциях.
